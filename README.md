# 🧩 Text-to-Extension Platform

A full-stack MERN application that lets users generate VS Code extensions from plain text descriptions. Includes user authentication, a management dashboard, file preview, and ZIP download.

---

## 🗂️ Project Structure

```
text-to-vscode/
├── backend/              # Express.js + MongoDB API
│   ├── controllers/      # Business logic
│   ├── middleware/       # JWT auth middleware
│   ├── models/           # Mongoose schemas
│   ├── routes/           # API routes
│   ├── server.js         # Entry point
│   └── .env.example
└── frontend/             # React.js SPA
    ├── public/
    └── src/
        ├── components/   # Reusable components
```

## ⚙️ Setup Instructions

### Prerequisites
- Node.js v18+
- MongoDB (local or Atlas)
- npm or yarn

---

### 1. Backend Setup

```bash
cd backend
npm install
cp .env.example .env
```

Edit `.env`:
```env
PORT=5000
MONGODB_URI=mongodb://localhost:27017/text-to-vscode
JWT_SECRET=your_super_secret_key_here
JWT_EXPIRE=7d
NODE_ENV=development
```

Start the server:
```bash
npm run dev     # Development (with nodemon)
npm start       # Production
```

Server runs at: `http://localhost:5000`

---

### 2. Frontend Setup

```bash
cd frontend
npm install
npm start
```

App runs at: `http://localhost:3000`

The React app proxies API calls to `http://localhost:5000` automatically.

## 🔌 API Endpoints

### Auth
| Method | Route | Description |
|--------|-------|-------------|
| POST | `/api/auth/register` | Register new user |
| POST | `/api/auth/login` | Login |
| GET | `/api/auth/me` | Get current user |
| PUT | `/api/auth/profile` | Update profile |

### Extensions
| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/extensions` | List all extensions (paginated) |
| POST | `/api/extensions` | Generate new extension |
| GET | `/api/extensions/stats` | Dashboard stats |
| GET | `/api/extensions/:id` | Get single extension |
| PUT | `/api/extensions/:id` | Update extension |
| DELETE | `/api/extensions/:id` | Delete extension |
| GET | `/api/extensions/:id/download` | Download as ZIP |

## 🚀 Features

- **User Auth** — JWT-based register/login/logout with bcrypt password hashing
- **Text-to-Extension** — Generate VS Code extension files from a text prompt
- **Dashboard** — Stats: total extensions, ready, failed, downloads, by category
- **File Preview** — Tab-based viewer for all generated files (package.json, extension.js, README.md, etc.)
- **Download as ZIP** — Download complete extension project as a `.zip`
- **Search & Filter** — Filter by category, status, or search by name/description
- **Pagination** — Backend paginated extension listing
- **Rate Limiting** — 100 req/15min per IP
- **Helmet** — HTTP security headers

## 🤖 Adding AI Generation (Optional)

The `extensionGenerator.js` currently uses template-based generation. To plug in OpenAI:

```bash
npm install openai
```

In `backend/controllers/extensionGenerator.js`, replace the `generateExtensionFiles` function body with an OpenAI API call:

```js
const OpenAI = require('openai');
const client = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

const response = await client.chat.completions.create({
  model: 'gpt-4o',
  messages: [
    { role: 'system', content: 'You are a VS Code extension expert...' },
    { role: 'user', content: `Generate a VS Code extension for: ${prompt}` }
  ]
});
```
